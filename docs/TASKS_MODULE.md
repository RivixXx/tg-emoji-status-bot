## 🚀 Быстрый старт

### 1. Выполнить миграцию БД

Открой [Supabase Dashboard](https://supabase.com/dashboard) → SQL Editor и выполни:

```sql
-- Файл: docs/init_tasks.sql
```

Или через CLI:

```bash
psql -h db.xxx.supabase.co -U postgres -d postgres -f docs/init_tasks.sql
```

### 2. Зарегистрировать команды в боте

В файле `main.py` добавь:

```python
from skills.task_commands import register_task_commands

# ... в функции main() ...
register_task_commands(bot, MY_ID)
```

### 3. Перезапустить бота

```bash
sudo systemctl restart tg-emoji-status-bot
```

### 4. Проверить работу

```
/tasks              → Показать все задачи
/task create Тест   → Создать задачу
/task               → Справка по командам
```

## 🗄️ База данных

### Таблицы

#### `projects`
- `id` — ID проекта
- `user_id` — ID владельца
- `name` — Название
- `description` — Описание
- `status` — active/paused/completed/archived
- `priority` — low/medium/high/urgent
- `deadline` — Дедлайн
- `metadata` — JSON с доп. данными

#### `tasks`
- `id` — ID задачи
- `user_id` — ID владельца
- `project_id` — Ссылка на проект
- `title` — Заголовок
- `description` — Описание
- `status` — todo/in_progress/review/done/cancelled
- `priority` — low/medium/high/urgent
- `due_date` — Дедлайн
- `estimated_hours` — Оценка времени
- `parent_task_id` — Для подзадач
- `tags` — Массив тегов

#### `sprints`
- `id` — ID спринта
- `user_id` — ID владельца
- `name` — Название
- `start_date` — Дата начала
- `end_date` — Дата окончания
- `goal` — Цель спринта
- `status` — planning/active/completed/archived

#### `sprint_tasks`
- `sprint_id` — ID спринта
- `task_id` — ID задачи
- Связь многие-ко-многим

#### `daily_goals`
- `id` — ID записи
- `user_id` — ID владельца
- `date` — Дата
- `goals` — Массив целей (до 10)
- `completed` — Массив булевых значений
- `evening_review` — Вечерний обзор
- `mood` — Настроение
- `productivity_score` — Оценка 1-10

#### `task_comments`
- `id` — ID комментария
- `task_id` — ID задачи
- `user_id` — ID автора
- `comment` — Текст

#### `task_history`
- `id` — ID записи
- `task_id` — ID задачи
- `action` — created/updated/status_changed/deleted
- `old_value` — JSON старых данных
- `new_value` — JSON новых данных

## 📊 Метрики

### Полезные SQL запросы

```sql
-- Задачи по статусам
SELECT status, COUNT(*) 
FROM tasks 
WHERE user_id = 582792393 
GROUP BY status;

-- Просроченные задачи
SELECT title, due_date 
FROM tasks 
WHERE due_date < NOW() 
AND status != 'done';

-- Прогресс проекта
SELECT 
    p.name,
    COUNT(t.id) as total,
    COUNT(t.id) FILTER (WHERE t.status = 'done') as completed,
    ROUND(COUNT(t.id) FILTER (WHERE t.status = 'done') * 100.0 / COUNT(t.id), 2) as percent
FROM projects p
LEFT JOIN tasks t ON p.id = t.project_id
GROUP BY p.id, p.name;
```

## 🐛 Troubleshooting

### Ошибка: "task not found"

**Причина:** Задача не найдена или принадлежит другому пользователю.

**Решение:**
```bash
# Проверь ID задачи
/tasks all

# Проверь владельца в БД
SELECT * FROM tasks WHERE id = 15;
```

### Ошибка: "duplicate key"

**Причина:** Задача с таким именем уже существует в спринте.

**Решение:**
```bash
# Удали дубликат
DELETE FROM sprint_tasks WHERE sprint_id = 1 AND task_id = 15;
```

### Задачи не отображаются

**Причина:** Не выполнена миграция БД.

**Решение:**
```bash
# Выполни миграцию
psql -f docs/init_tasks.sql
```
