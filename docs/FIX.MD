# Bugfix Log (21.02.2026)

## [FIXED] Бесконечный цикл утренних напоминаний
- **Проблема**: Напоминания отправлялись каждую минуту после срабатывания.
- **Решение**: Добавлена принудительная деактивация (`is_active = False`) и сохранение состояния в БД сразу после отправки сообщения.

## [FIXED] Ошибка смены эмодзи-статуса (Invalid document file)
- **Проблема**: Telegram отклонял устаревшие ID эмодзи, вызывая ошибки в логах.
- **Решение**: 
    - Обновлены ID для `weekend` и `work` на верифицированные рабочие значения.
    - Добавлен "Детектор ID" (`id [emoji]` в Избранном) для быстрого поиска кодов кастомных эмодзи.
    - Внедрена блокировка повторных попыток смены статуса при ошибке до следующего временного интервала.

## [FIXED] Ложные срабатывания подтверждения здоровья
- **Проблема**: Любое слово "да" или "готово" воспринималось как отчет об уколе.
- **Решение**: Логика подтверждения теперь активируется только если в системе есть активное (незакрытое) напоминание о здоровье на текущий день.

## [FIXED] Напоминания в "Избранном" вместо лички
- **Проблема**: Сообщения отправлялись через UserBot самому себе.
- **Решение**: Переключен клиент отправки напоминаний на `bot_client`.

## [FIXED] Ошибка AttributeError в кнопках
- **Проблема**: `event.message` был недоступен в CallbackQuery.
- **Решение**: Добавлен явный вызов `await event.get_message()`.
