# 🎯 Система проактивных триггеров

**Версия:** 1.0
**Дата:** Март 2026
**Статус:** ✅ Готово к использованию

---

## 📋 Обзор

Система проактивных уведомлений и напоминаний, которая делает Karina AI **активным помощником**, а не просто реактивным ботом.

Триггеры автоматически активируются по расписанию и побуждают пользователя к действию.

---

## ⏰ Расписание триггеров

| Время | Триггер | Описание |
|-------|---------|----------|
| **7:00** | Утреннее планирование | План на день, встречи, дедлайны |
| **9:00** | Просроченные задачи | Проверка должников |
| **12:00** | Застрявшие задачи | Задачи 3+ дня в todo |
| **13:00** | Обед | Напоминание поесть |
| **:30** | Перерывы | Каждый час в 30 минут |
| **:00** | Дедлайны | Проверка каждый час |
| **20:00** | Вечерний обзор | Итоги дня, рефлексия |

---

## 🌅 Утреннее планирование (7:00)

### Что делает:

1. **Показывает события на сегодня** из календаря
2. **Напоминает о дедлайнах** на сегодня
3. **Показывает просроченные задачи**
4. **Показывает активный спринт** с прогрессом
5. **Даёт совет Карины** в зависимости от загруженности

### Пример сообщения:

```
☀️ Доброе утро, Михаил!

📅 План на сегодня:

📍 Встречи:
  • 10:00 — Команда
  • 14:00 — Клиент

⚠️ Дедлайны на сегодня:
  🔴 Подготовить отчёт
  🟡 Отправить письмо

🏃 Активный спринт: Март 2026
  ⏳ Осталось дней: 5
  📊 Прогресс: 60%

💡 Совет Карины:
Сегодня плотный день! Сфокусируйся на 1-3 самых важных задачах.

[🎯 Цели на сегодня] [📋 Показать задачи] [🏃 Спринт]
```

---

## 🌙 Вечерний обзор (20:00)

### Что делает:

1. **Показывает выполненные задачи** за день
2. **Напоминает о завершении спринта** если скоро конец
3. **Предлагает записать итоги дня**
4. **Планирует завтрашний день**

### Пример сообщения:

```
🌆 Вечер на дворе!

✅ Сегодня выполнено:
  • Подготовить отчёт
  • Позвонить клиенту
  • Код ревью

Всего: 3 задачи(и)

⏰ Спринт заканчивается завтра!
Проверь что всё успеваешь.

📝 Вечерний обзор:

Как прошёл твой день?
Что получилось хорошо?
Что можно улучшить завтра?

[📝 Записать итоги] [🎯 Цели на завтра] [😴 Всё, спать!]
```

## ☕ Перерывы (каждый час в :30)

### Работает:
- Только в будни (пн-пт)
- Только в рабочее время (9:00-18:00)

### Примеры сообщений:

```
☕ Время перерыва!

5 минут отдыха повысят продуктивность. Разомнись!

[✅ Уже иду!] [⏰ Позже (15 мин)]
```

```
🧘 Перерыв!

Посмотри вдаль, разомни шею. 5 минут — и снова в бой!
```

```
👀 Отдых для глаз!

Посмотри в окно 20 секунд. Береги зрение!
```

## 🏗️ Архитектура

### Модуль: `brains/triggers.py`

**Классы:**
- `TriggerState` — состояние триггеров (защита от дублирования)

**Функции:**
- `morning_planning_trigger()` — утреннее планирование (7:00)
- `evening_review_trigger()` — вечерний обзор (20:00)
- `deadline_reminder_trigger()` — дедлайны (каждый час)
- `stuck_task_trigger()` — застрявшие задачи (12:00)
- `break_reminder_trigger()` — перерывы (:30)
- `lunch_reminder_trigger()` — обед (13:00)
- `overdue_tasks_trigger()` — просроченные задачи (9:00)
- `start_triggers_loop()` — главный цикл

**Защита от дублирования:**

Каждый триггер проверяет:
1. **Время активации** (например, 7:00 ± 1 минута)
2. **Дата последней отправки** (чтобы не дублировать в тот же день)
3. **Минимальный интервал** (не чаще раза в час для некоторых триггеров)

## 🐛 Troubleshooting

### Триггер не срабатывает

**Причина 1:** Уже отправлялось сегодня

**Решение:**
```python
# Проверить состояние
from brains.triggers import trigger_state
print(trigger_state.last_morning_planning)  # Дата последней отправки
```

**Причина 2:** Не то время

**Решение:**
```python
# Проверить время
from datetime import datetime, timezone, timedelta
now = datetime.now(timezone(timedelta(hours=3)))
print(f"Время: {now.hour}:{now.minute}")
```

**Причина 3:** Выходной

**Решение:**
```python
# Некоторые триггеры не работают в выходные
print(f"День недели: {now.weekday()}")  # 0-пн, 5-сб, 6-вс
```

### Дублирование сообщений

**Причина:** Несколько триггеров сработали одновременно

**Решение:**
- Проверить `trigger_state` на корректность обновления
- Увеличить интервал между проверками

### Кнопки не работают

**Причина:** Callback не зарегистрирован

**Решение:**
```python
# Проверить что callback обработчик зарегистрирован
```
