## 2026-03-08 — Исправления и улучшения

### 🔧 Fix: Баннеры — сохранять InputPhoto id/access_hash вместо pickle
**Коммит:** `505d99f`

**Проблема:**
- При сохранении баннеров использовался `pickle` для сериализации объектов Telegram Media
- При загрузке эти объекты не могли быть корректно восстановлены
- Ошибка: `Failed to convert gASVEA... to media. Not an existing file, an HTTP URL or a valid bot-API-like file ID`

**Решение:**
- Теперь сохраняются только `id`, `access_hash` и `file_reference` фото в JSON-формате
- При загрузке создаётся объект `InputPhoto`, который Telegram понимает корректно
- Удалён `banners_cache.json` со старым форматом кэша

**Файлы:**
- `brains/vpn_logic.py` — изменены функции `preload_banners()` и `get_cached_banner()`
- `DEPLOY.md` — добавлена инструкция по развёртыванию

## 🐛 Известные особенности

1. **Дублирование вызовов инструментов AI** — Mistral AI может возвращать 2 одинаковых `tool_calls` в одном ответе (например, `get_weather_info` вызывается дважды). Это не баг, код обрабатывает каждый вызов корректно.

2. **Первый запуск** — загрузка баннеров занимает ~45 секунд. Последующие запуски мгновенные (кэш в `banners_cache.json`).

3. **Таймауты Supabase** — при медленном соединении возможны таймауты (>60с). Обработано в `search_memories`.
