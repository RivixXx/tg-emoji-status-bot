# 🚀 Инструкция по развёртыванию

## После деплоя на сервере

### 1. Удалите старый кэш баннеров (если существует)
```bash
rm -f ~/tg-emoji-status-bot/banners_cache.json
```

### 2. Перезапустите бота
```bash
# Если используете systemd
sudo systemctl restart tg-emoji-status-bot

# Или если используете screen/tmux
# Просто перезапустите процесс
```

### 3. Проверьте логи
```bash
tail -f ~/tg-emoji-status-bot/bot.log
```

**Ожидаемая последовательность в логах:**
```
🖼 Проверка кэша баннеров на диске...
⏳ ЗАГРУЖАЮ 'menu' ОДИН РАЗ НАВСЕГДА (займет ~45 сек)...
✅ Баннер 'menu' успешно загружен!
💾 Кэш баннеров сохранен в banners_cache.json! Больше загрузок не будет.
⚡ Все баннеры найдены на диске! Загрузка мгновенная.
```

## ⚠️ Важно!

- **Первый запуск** займет ~45 секунд на загрузку баннеров в Telegram
- После этого `banners_cache.json` будет сохранён и **повторная загрузка не потребуется**
- При следующем запуске бот мгновенно загрузит баннеры из кэша

## 🔧 Если баннеры не работают

1. Проверьте, что файлы баннеров существуют:
   ```bash
   ls -la ~/tg-emoji-status-bot/banners/
   ```

2. Проверьте права доступа к файлам:
   ```bash
   chmod 644 ~/tg-emoji-status-bot/banners/*.jpg
   ```

3. Если кэш повредился — удалите и перезапустите:
   ```bash
   rm ~/tg-emoji-status-bot/banners_cache.json
   sudo systemctl restart tg-emoji-status-bot
   ```

## 📝 Что было исправлено

**Проблема:** При сохранении баннеров использовался `pickle`, который сериализовал объекты Telethon. При загрузке эти объекты не могли быть корректно восстановлены.

**Решение:** Теперь сохраняются только `id`, `access_hash` и `file_reference` фото в JSON-формате. При загрузке создаётся объект `InputPhoto`, который Telegram понимает корректно.
